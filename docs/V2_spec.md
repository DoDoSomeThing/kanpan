## 0. 四大原則（刻死，凡違反一律砍）

1. **系統不預測未來** — 不輸出「會漲/會跌」。
2. **顯示歷史，不輸出信心** — 給樣本/勝率/報酬，不給「我覺得」。
3. **無回測 = 無意見** — 沒數據就寫「無回測數據」，不補幻想數字。
4. **風控優先於訊號** — 主角是持倉風控，不是買點神判。

對齊既有事實：vp_score / 判讀燈號**已驗證無勝率 edge**（90-100 桶勝率 45%、輸低分桶），故任何「分數→下注大小」一律禁止。唯一驗過的真 edge = **trail 出場（−4% 硬停損 / 高點回落 8%）**。

## Phase 1 — L3 持倉風控（先做，最高價值）

### 資料模型 `positions.json`（新增，本機，gitignore）
```json
{
  "open": {
    "2356": {
      "entry_price": 68.6,
      "shares": 0.5,
      "entry_date": "2026-06-18",
      "peak_price": 73.2,
      "note": ""
    }
  },
  "closed": [
    {
      "sid": "2356", "entry_price": 68.6, "exit_price": 67.3,
      "entry_date": "2026-06-18", "exit_date": "2026-07-02",
      "exit_reason": "trail", "return_pct": -1.9, "hold_days": 10
    }
  ]
}
```
- 支援**多檔**（`open` key = sid）。
- `peak_price` = 進場後每日 high 的累積 max，**跨日持久化**（每次刷新用 `max(peak_price, 今日high)` 更新寫回）。
- **決議②：記已實現出場歷史** → `closed` 陣列。供日後自我檢討「賣太差了嗎」。出場時把該檔從 `open` 搬進 `closed`。

### 計算邏輯
| 項目 | 公式 |
|---|---|
| 未實現% | `(現價 − entry)/entry × 100` |
| 硬停損 | `entry × (1 − 0.04)`（固定，從進場價算） |
| Trail 保護價 | `peak × (1 − 0.08)` |
| **生效出場價** | `max(硬停損, Trail保護價)` ← 取較高者，面板只秀這條 |
| 距觸發% | `(現價 − 生效出場價)/現價 × 100` |
| 狀態燈 | 🟢 正常(距觸發 > 2%)｜🟡 接近(0~2%)｜🔴 已觸發(現價 ≤ 生效出場價) |

> 早期 peak 未漲高時硬停損當地板；peak 漲上去後 Trail 超過硬停損才接手。面板標「目前由 硬停損／Trail 生效」。

### UI 輸出（範例）
```
持倉 2356 英業達
進場 68.6 × 0.5張｜現價 71.3｜未實現 +3.9%
生效出場：67.3（Trail，高點73.2−8%）
距觸發 −5.6%　🟢 正常持有
```

### 整合點
- 新增 `position.py`（讀寫 positions.json + 算風控）。
- `api.py` 加 `/position` 端點 + `/panel` 回傳該檔風控（若有持倉）。
- `web/index.html` + `extension/content.js` 加持倉卡（最上方，風控優先）。
- CLI 加 `--pos` 或獨立 `position.py` 指令查。

## Phase 3 — L2 規則化劇本引擎（最後做，防呆優先）

### 固定模板（禁任意組合，擋資料探勘）
| 模板 | 精確條件（草案，待定稿） |
|---|---|
| 突破型 | 收盤 > 近20日高 且 量 > 1.2×MV20 |
| 跌破型 | 收盤 < 近20日低 |
| 回測型 | 站回 MA20（前一根 < MA20、本根 ≥ MA20）且 量 ≥ MV20 |

> 只此三型，**不開放使用者自由湊條件**（自由湊 = 保證探勘到假 edge）。

### 回測方法
- 資料 = `kline_deep.json`（2021-06~2024-12）。
- **前後分段**：前半段 = 訓練、後半段 = 驗證，各算條件成立後 20 日勝率/平均報酬/最大回撤。
- 重用 `research/score_history.py` 的取樣框架。

### 顯示
```
劇本 A 突破型｜條件：收盤>近20高 且 量>1.2×MV20
樣本 74｜訓練勝率 57%｜驗證勝率 53%｜平均 +4.2%｜最大回撤 −5.8%
規則：條件成立 → 執行預設策略（進場後套 Phase1 風控）
```

### 防呆（三條，缺一不可）
```
if n < 30:                      顯示「樣本不足，不給勝率」
if 訓練勝率 − 驗證勝率 > 10pp:    顯示「過擬合警告」
if 無樣本外:                     顯示「未驗證」
```

### L2.5 — 不交易原因（把「今天別做」做成功能）
```
不交易原因：
☑ 樣本不足
☑ 型態無 edge（勝率<50%）
☐ 過擬合
☐ 條件未成立
```
> 心理準備：照既有驗證，三模板大概率跑出勝率 45~50% → 引擎價值 = **誠實證明多數型態沒料、勸退亂進場**，非找神劇本。
